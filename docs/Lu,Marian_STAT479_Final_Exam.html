1 Data Preprocessing

1.1 Data Selection

Introduction

The dataset is provided by Kaggle and contains images of cells segmented from microscopic images. These cells are representative of images in the real-world because they contain some staining noise and illumination errors, although these errors have largely been fixed in the course of acquisition. In total there are 15,135 images from 118 patients, which are categorized into two labelled classes: normal cells, and leukemia blasts.

1.2 Preprocessing Images

All the images are 450x450 pixels. This is rather large, so they will be scaled down to 128x128 pixels for more optimal processing in later steps of training. The original images are also in color, so a grayscale will be applied to simply the network architecture. The images are organized in the following folder structure: fold_n (representing the patient cohort) -> [all (normal cells), hem (leukemia blasts)]. For more lightweight processing, instead of using the full dataset of over 15,000 images, I will be using fold_0 as training data, and splitting fold_1 for testing and validation data.

In[95]:
def read_images(fold_n, img_class):
    dirpath = os.path.join('data', fold_n, img_class)
    all_images = []
    for i in os.listdir(dirpath):
        image_path = os.path.join('data', fold_n, img_class, i)
        img = io.imread(image_path, as_gray=True)
        all_images.append(cv2.resize(img, (128, 128)))

    X = np.array(all_images)

    return X

2 Model Design and Implementation

2.1 Defining CNN

The structure of a CNN consists of the following:

  • Input Layer - Images are read in to the network, usually as numerical arrays.
  • Convolutional Layers - Filters are used to extract features from the input image. This is where the network recognizes general patterns.
  • Pooling Layers - These are used to decrease dimensions of feature maps to reduce complexity and reduce overfitting.
  • Fully Connected Layers - This layer connects all neurons from the previous layer and is able to recognize and respond to high-level features
  • Output Layer - Returns the output of the neural network. In this case, it will be returning either 0 or 1, since this is a binary classification task.

3.1 ROC Curve and AUC

The Receiver Operator Characteristic (ROC) Curve is a metric for evaluating binary classificaiton models. It shows the performance of the model at all thresholds. A perfect ROC curve would resemble a right angle in upper left diagonal of the plot. The ideal AUC (area under curve) would be 1, since it means that the model has 100% accuracy.

def plot_confusion_matrix(cm, classes, normalize=False, title='Confusion matrix', cmap=plt.cm.Blues):
    """ 
    This funciton prints and plots the confusion matrix.
    Normalization can be applied by setting 'normalize=True'.
    """
    if normalize:
        cm = cm.astype('float')/cm.sum(axis=1)[:, np.newaxis]
        print("Normalized confusion matrix")
    else:
        print('Confusion matrix, without normalizaiton')
    print(cm)

    plt.imshow(cm, interpolation='nearest', cmap=cmap)
    plt.title(title)
    plt.colorbar()
    tick_marks = np.arange(len(classes))
    plt.xticks(tick_marks, classes, rotation=45)
    plt.yticks(tick_marks, classes)

    fmt = '.2f' if normalize else 'd'
    thresh = cm.max()/2.
    for i, j in itertools.product(range(cm.shape[0]), range(cm.shape[1])):
        plt.text(j, i, format(cm[i,j], fmt), horizontalalignment="center",
                 color="white" if cm[i,j] > thresh else "black")
    plt.tight_layout()
    plt.ylabel('True')
    plt.xlabel('Predicted')
    plt.show()
